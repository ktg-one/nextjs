<task type="auto">
  <name>Task 2: Touch device detection — disable cursor and glow, cap shader DPR</name>
  <files>src/components/CustomCursor.tsx, src/components/SdfBackground.tsx</files>
  <action>
In CustomCursor.tsx:
1. At component mount (useEffect), check `'ontouchstart' in window || navigator.maxTouchPoints > 0`
2. If touch device: return null (render nothing — no cursor div, no ambient glow div)
3. Also set `document.body.style.cursor = 'auto'` on touch devices to restore native cursor
4. This completely removes both the custom cursor AND the ambient glow from the DOM on touch

In SdfBackground.tsx:
1. In the renderer setup, detect mobile via `'ontouchstart' in window || window.innerWidth <= 640`
2. If mobile: cap DPR at 1 instead of 2 — `Math.min(window.devicePixelRatio, 1)`
3. If desktop: keep existing cap at 2 — `Math.min(window.devicePixelRatio, 2)`
4. Update the u_pixelRatio uniform accordingly
5. On resize, re-evaluate the DPR cap (user might rotate device or resize browser)
6. Add `prefers-reduced-motion` media query check: if user prefers reduced motion, pause the RAF loop and show static frame (render once, then stop animating)

Why DPR 1 on mobile: Pitfall 6 (PITFALLS.md) — SDF shader at full DPR causes thermal throttling within 30-60s on mobile. iOS Safari aggressively kills WebGL contexts under thermal pressure.
  </action>
